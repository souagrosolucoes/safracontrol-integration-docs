Fluxo de Criação de Pedido

Guia passo a passo com a ordem correta de criação dos recursos necessários para registrar um pedido na API do SafraControl.

Visão Geral das Dependências

O pedido (order) depende de outros recursos que precisam existir previamente. O diagrama abaixo mostra todas as dependências obrigatórias e opcionais: 

  ┌─────────────┐    ┌──────────────────┐
  │  Produtos   │    │    Contatos       │
  │ /products   │    │                  │
  │             │    │  /customers      │ ← Cliente (obrigatório)
  │ /product-   │    │  /branch-offices │ ← Filial  (obrigatório)
  │  varieties  │    │  /sellers        │ ← Vendedor (opcional)
  └──────┬──────┘    │  /suppliers      │ ← Fornecedor (opcional)
         │           │  /withdrawal-    │ ← Local de  (opcional)
         │           │   places         │   Retirada
         │           └────────┬─────────┘
         │                    │
         └──────────┬─────────┘
                    ▼
             ┌─────────────┐
             │   /orders   │
             │   Pedido    │
             └──────┬──────┘
                    │
                    ▼
            ┌──────────────┐
            │ /order-items │
            │ Itens do     │
            │ Pedido       │
            └──────────────┘
ℹ️
Todos os recursos utilizam o campo externalId como identificador externo. Use o ID do seu sistema como valor — a API usa esse campo para evitar duplicatas e para referências entre recursos.

Passo a Passo

1
Criar Produto
POST/products

Cada item do pedido referencia um produto. Crie os produtos antes de criar os itens do pedido. Se o produto já existir, você pode usar a opção de upsert (?upsert=true na query).

Campos principais

Campo Tipo Descrição
externalId string obrigatório Identificador único no seu sistema
name string opcional Nome do produto
unit string opcional Unidade de medida (ex: KG, SC)
group object opcional Grupo do produto — { "externalId": "..." }
Exemplo de requisição
// POST /products
{
  "externalId": "PROD-001",
  "name": "Soja",
  "unit": "SC",
  "weight": 60
}
2
Criar Contatos
POST/contacts (e subtipos)

O pedido precisa de pelo menos um cliente e uma filial. Outros tipos de contato são opcionais, mas frequentemente usados. Crie cada tipo de contato pelo seu endpoint específico:

Cliente obrigatório
POST /customers
Filial obrigatório
POST /branch-offices
Vendedor opcional
POST /sellers
Fornecedor opcional
POST /suppliers
Local de Retirada opcional
POST /withdrawal-places
Local de Entrega opcional
POST /deposits
💡
Todos os subtipos compartilham a mesma estrutura base com externalId obrigatório. O endpoint POST /contacts cria contatos genéricos; use os endpoints específicos para garantir que o tipo correto seja associado automaticamente.

Campos principais (todos os subtipos)

Campo Tipo Descrição
externalId string obrigatório Identificador único no seu sistema
name string opcional Nome / razão social do contato
document string opcional CPF ou CNPJ
Exemplo — Criar cliente
// POST /customers
{
  "externalId": "CLI-100",
  "name": "Fazenda Boa Vista Ltda",
  "document": "12.345.678/0001-90"
}
Exemplo — Criar filial
// POST /branch-offices
{
  "externalId": "FILIAL-01",
  "name": "Filial São Paulo"
}

Adicionar endereço ao contato (opcional)

Após criar um contato, você pode associar endereços usando o endpoint aninhado: 

// POST /contact/{contactExternalId}/addresses
{
  "externalId": "END-100-1",
  "street": "Av. Paulista",
  "number": "1000",
  "city": "São Paulo",
  "state": "SP"
}
3
Criar Pedido
POST/orders

Com produtos e contatos criados, você já pode criar o pedido. As referências são feitas pelo externalId de cada recurso.

Campos principais

Campo Tipo Descrição
externalId string obrigatório Identificador único do pedido no seu sistema
customer object obrigatório Referência ao cliente — { "externalId": "CLI-100" }
branchOffice object obrigatório Referência à filial — { "externalId": "FILIAL-01" }
freightType string obrigatório Tipo de frete (ex: CIF, FOB)
dueDate string obrigatório Data de vencimento — formato ISO 8601
period object obrigatório Período de safra — { "externalId": "..." }
seller object opcional Referência ao vendedor — { "externalId": "VEND-01" }
regional object opcional Referência à regional — { "externalId": "..." }
Exemplo de requisição
// POST /orders
{
  "externalId": "PED-2024-001",
  "freightType": "CIF",
  "dueDate": "2024-12-31",
  "customer": { "externalId": "CLI-100" },
  "branchOffice": { "externalId": "FILIAL-01" },
  "period": { "externalId": "SAFRA-2024" },
  "seller": { "externalId": "VEND-01" }
}
⚠️
Os campos customer, branchOffice e period precisam referenciar recursos que já existam na API. Se a referência não for encontrada, a criação retornará erro 400.
4
Criar Itens do Pedido
POST/order-items

Por último, crie os itens vinculando cada produto ao pedido criado no passo anterior. Um pedido pode ter múltiplos itens — envie uma requisição para cada item.

Campos principais

Campo Tipo Descrição
externalId string obrigatório Identificador único do item no seu sistema
order object opcional Referência ao pedido — { "externalId": "PED-2024-001" }
product object opcional Referência ao produto — { "externalId": "PROD-001" }
quantity number opcional Quantidade do item
unitValue number opcional Valor unitário
withdrawalPlace object opcional Local de retirada — { "externalId": "RETIRADA-01" }
Exemplo de requisição
// POST /order-items
{
  "externalId": "ITEM-2024-001-1",
  "order": { "externalId": "PED-2024-001" },
  "product": { "externalId": "PROD-001" },
  "quantity": 100,
  "unitValue": 150.00,
  "status": "active"
}

Resumo da Ordem de Criação

  1. Crie os grupos de produtos (POST /groups) se necessário
  2. Crie os produtos (POST /products) — use ?upsert=true para atualizar se já existir
  3. Crie os períodos (POST /periods) se necessário
  4. Crie a filial (POST /branch-offices)
  5. Crie o cliente (POST /customers)
  6. Crie os demais contatos opcionais: vendedor (/sellers), fornecedor (/suppliers), local de retirada (/withdrawal-places)
  7. Crie o pedido (POST /orders)
  8. Crie os itens do pedido (POST /order-items)
💡
Use ?upsert=true nas requisições para criar ou atualizar um recurso em uma única chamada, sem precisar verificar se ele já existe. Isso simplifica a lógica de sincronização do seu sistema.
